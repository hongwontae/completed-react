1. 스타일 컴포넌트 설치방법
=> npm install styled-components


2. CSS-in-JS
2-1. 스타일을 JS 파일 내에서 정의하고, 조건부 로직, 변수 등을 함께 사용할 수 있는 구조입니다.
2-2. JS 문법 안에 CSS를 사용
2-3. 컴포넌트 단위로 스타일 작성
2-4. 동적 스타일 가능
2-5. 대표적인 라이브러리로 styled-components, emotion, Sitiches, goober...


3. CSS-in-JS 단점
3-1. 런타임 성능 저하
=> CSS가 JS 실행 중에 동적으로 생성되어, 초기 렌더링시 오버헤드 발생할 수 있습니다.
3-2. 디버깅이 불편합니다.
3-3. 스타일 분리의 원칙에 위배됩니다.
3-4. 스타일이 JS 번들에 포함됨으로, 번들 파일 크기가 커질 수 있습니다.
