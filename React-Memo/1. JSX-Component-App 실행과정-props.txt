1. JSX
1-1. HTML안에 JS 문법을 사용할 수 있는 문법 확장입니다.
1-2. {}를 사용해서 JS를 추가할 수 있습니다.


2. Components
2-1. JSX를 반환하는 함수입니다.
2-2. JSX를 반환할 떄는 JSX가 하나의 루트여야 합니다. (하나의 태그로 모든 태그가 덮여져 있어야 합니다.)
2-3. 대문자로 시작해야 하고 다른 파일에 IMPORT 위해서 Export가 필요합니다.


3. React App 실행과정
3-1. 브라우저에 React App이 도달한다는 것은 index.html을 전달한다는 뜻입니다.
(해당 HTML은 거의 비어있는 상태입니다.)
3-2. index.html의 script 태그의 src를 통해 번들링된 JS 파일을 HTTP 요청으로 가져옵니다.
(컴포넌트, 라우팅 로직, 상태 관리 코드.. 등등이 존재합니다.)
3-3. 처음 실행되는 JS 파일은 main.jsx이며 여기서 트리구조로 React UI가 만들어지고 화면에 렌더링됩니다.


4. props
4-1. 컴포넌트에 데이터를 전달하는 개념을 의미합니다.
4-2. 전달할 데이터는 단순 데이터 뿐만 아니라 동적인 데이터도 가능합니다.
4-3. 전달할 컴포넌트에서 JSX에 KEY={VALUE}로 전달하고 받을 컴포넌트에서 props로 받아서
 props.KEY로 사용하면 됩니다. (받을 컴포넌트에서 객체 구조 분해를 통해 {KEY, KEY, KEY}로 받을 수 있습니다.)
4-4. 받을 컴포넌트에서 전달할 컴포넌트의 props가 도달하지 않았다면 기본값을 사용하여 대비할 수 있습니다.
(function ChildComponent({caption, type = "submit"}))
