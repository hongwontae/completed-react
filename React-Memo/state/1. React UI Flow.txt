1. React UI Flow
1-1. 리액트 앱에 UI를 변경한다는 것은 컴포넌트를 재실행한다는 것입니다.
1-2. 일반적인 함수로는 컴포넌트를 재실행할 수 없고 권한도 없습니다.
1-3. 리액트는 상태라는 개념을 활용해 상태에 의미를 넣어 UI를 설계하고 
 상태 변경 함수를 사용하여 컴포넌트를 재실행하여 UI를 변경할 수 있습니다.


2. 컴포넌트 재실행
2-1. 리액트에서 컴포넌트를 재실행한다는 것은 JSX를 다시 실행한다는 것이 아닙니다.
2-2. 상태변경함수실행(para) -> 컴포넌트가 재실행 component() -> return 문의 위의 일반 코드들 실행 ->
 상태값 재계산 -> 변경된 요소가 있을 떄 가상 돔을 새로 생성 -> 현 돔, 가상 돔의 차이만큼만 갱신
