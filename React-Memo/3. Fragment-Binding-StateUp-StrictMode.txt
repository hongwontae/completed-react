1. React Fragment
=> 컴포넌트의 JSX 반환은 형제 형태를 이루고 있는 것이 아닌 하나의 부모안에 모든 JSX가 있어야 합니다.
=> <main>....<main/>
=> 하지만 이런 부모 JSX가 CSS 오류, HTML 구조의 복잡성 증가 등.. 여러 불편함을 야기합니다.
=> 이 떄 <React.Fragment><React.Fragment/>로 덮거나 <></> 빈 태그를 사용하여 해결할 수 있습니다.


2. 양방향 바인딩
=> 상태와 입력 값이 서로 영향을 주는 구조를 의미합니다.
=> input 태그의 value는 state값을 주고 onchage에는 이벤트 핸들러를
  추가하여 e 객체를 받아서 처리하는 구조가 대표적입니다.


3. 상태 끌어올리기
=> 상태와 상태 변경 함수는 상위 컴포넌트에 정의했을 떄 하위 컴포넌트에서 함수를 호출하고 싶다면
 props drilling으로 상태 변경 함수, 함수를 전달하고 하위 컴포넌트가 호출하고 싶을 떄 
 호출하면 됩니다. (간단히 말해서 상태 변경 제어권이 하위 컴포넌트에게도 있다.)


4. React Strict Mode
4-1. <App> 컴포넌트를 감싸는 <StrictMode>가 존재합니다.
4-2. 역할
  1. React가 더 이상 권장하지 않거나 앞으로 제거될 API들을 사용하고 있다면 경고해줍니다.
  2. useEffect, useState.. 초기화 검증을 위해 두 번 호출합니다.
  => 개발 모드에서만 실행됩니다.
  3. 향후 React 기능 호환성을 확인합니다.
=> <StrictMode>는 오직 개발 모드에서만 동작하고 프로덕션에서는 완전히 무시됩니다.
